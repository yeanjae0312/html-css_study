14폼태그

폼 태그란? 사용자가 입력한 정보(아이디, 패스워드 등)를 서버로 전송하는 역할을 하며, 폼태그를 활용한 양식의 종류에는 로그인폼, 회원가입폼, 검색폼, 예약폼 등의 다양한 양식을 만들 수 있다.

양식은 form태그를 묶어서 작성하며 name(폼이름), method(전송방법), action(받을 서버주소) 속성이 반드시 들어가야 한다.

1. 폼태그의 종류

  • form : 입력 정보를 서버로 보낼 때
  • input : 글자를 입력하는 양식이나 버튼
  • select, option : 선택목록
  • textarea : 여러 줄의 텍스트 입력양식
  • label : 양식의 이름표
  • button : 버튼양식

1.GET : 소량의 데이터를 전송할 때 사용하며, 데이터 전송시 주소가 공개된다는 단점.(보안에 약함)
2.POST : 대량의 데이터를 전송할 때 사용하며 서버로 데이터 전송시 주소가 비공개로 전송되는 장점이 있다.(주로 많이 사용)
폼태그는 한번만 사용하면 된다.

1.텍스트박스 - 글자를 입력하기 위한 박스

2. 패스워드박스 - 패스워드나 공개되지 않는 데이터를 입력할 때

3. 라디오버튼 - 여러개의 항목중에 사용자가 원하는 항목을 선택하기 위한 양식. name속성을 사용하여 그룹핑 한다.

4. 체크박스 - 여러개의 항목중에 사용자가 원하는 항목만 다중선택할 수 있는 양식

취미 선택하기 :

5. 선택목록 - 사용자가 클릭하면 목록이 펼쳐지면서 선택하는 양식

쇼핑몰 제품선택, 통신사, 사이즈, 인원수, 년, 월, 일 선택 등에 활용

6. 다중텍스트 박스 - 한줄이 아닌 여러줄의 내용을 입력하고자 할 때 사용하는 양식이며, 주로 이메일쓰기, 게시판 글 작성하기, 문의하기 등에 활용할 수 있다.

7. 파일첨부 - 버튼을 클릭하면 파일을 첨부하기 위한 창이 활성화됨.

8. 전송버튼, 취소버튼, 일반버튼, 이미지버튼

전송버튼을 사용자가 입력한 내용을 서버로 전송하는 역할을 함.

취소하기 버튼은 사용자가 입력한 내용을 삭제함(화면이 리셋됨)

일반버튼은 '중복확인'이나 기타전송, 취소하기 이외의 버튼에 사용

이미지버튼 이미지를 삽입하여 문자대신에 보여주기 위한 버튼으로서 시각적인 표현이 가능하다.
